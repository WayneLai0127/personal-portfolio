## Thoughts

講者首先以“什麼是 DX“作為開場，這個詞在各大論壇討論串中經常被提及，然而，如何量化 DX 卻是一個難以回答的問題。作為開發者大會的開場，講者巧妙地提出了一個三角關係圖，雖然當時未以影像紀錄，但大致如下：

![dx-triangle](public/assets/aws-dev-day-2023/dx-metrix-simple.png)

這強而有力的圖形讓聽眾印象深刻，接著講者解釋這三個元素與 D 的關聯：

![dx-metrix](public/assets/aws-dev-day-2023/dx-metrix.jpg)

這三點分類又可進一步區分為主觀的感知與客觀的工作流數據，兩者綜合起來形成可視化的目標（KPI）。

## TLDR

TL;DR: 我們可以運用工具，例如 Code Catalyst（GitHub Copilot），搭配強化的 CICD（參見下圖左），以及 "Docs as Code" 的方式（參見下圖右），來強化 DX 體驗。在DX核心三角形的論點中，AWS 提供相應的工具，以加強開發者的體驗。
